# Databricks Spatial Types and Functions

Databricks provides **built-in spatial types** and **ST (spatial) and H3 (grid) functions** in Databricks SQL and Databricks Runtime 17.1+. This page gives a concise overview with runnable examples. Use these alongside GeoBrix for workflows that combine native Databricks geometry with GeoBrix raster/vector readers and GridX (e.g. BNG).

:::info Databricks Runtime required
Examples on this page are validated under `docs/tests-dbr/` and require **Databricks Runtime 17.1+** (or Databricks SQL with ST/H3 support). They will skip when run on open-source Spark.
:::

**References:**
- [GEOMETRY type](https://docs.databricks.com/aws/en/sql/language-manual/data-types/geometry-type) — Euclidean X,Y (and Z); SRID or ANY
- [GEOGRAPHY type](https://docs.databricks.com/aws/en/sql/language-manual/data-types/geography-type) — Lat/lon; SRID 4326 only
- [ST geospatial functions](https://docs.databricks.com/aws/en/sql/language-manual/sql-ref-st-geospatial-functions) — Import, export, measurements, predicates, overlay
- [H3 geospatial functions](https://docs.databricks.com/aws/en/sql/language-manual/sql-ref-h3-geospatial-functions) — Grid indexing with hexagons

## Combining with GeoBrix

- **Readers:** Use [GeoBrix readers](/docs/readers/overview) (GDAL, GeoJSON, Shapefile, etc.) to load vector/raster data; then use `st_geomfromwkb(geom_0)` or similar to get Databricks `GEOMETRY`/`GEOGRAPHY` for ST or H3.
- **GridX (BNG):** For British National Grid, use [GridX BNG functions](/docs/api/gridx-functions) (e.g. `bng_polyfill`, `bng_tessellate`). For a global hex grid, use Databricks H3.
- **RasterX:** Use [RasterX](/docs/api/rasterx-functions) for raster I/O and operations; combine with ST/H3 for zonal stats or grid aggregation in SQL.
